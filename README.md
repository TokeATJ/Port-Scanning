# Port-Scanning


## Overview

This excerise is focused on network reconnaissance and port scanning techniques using Hping3, Wireshark, NetBIOS enumeration, and Nmap. The objective was to identify live hosts, discover open ports, analyze packet behavior, perform UDP and TCP packet crafting, and observe network traffic generated during scanning activities.

## Learning Objectives

- Perform host discovery using ICMP packets
- Conduct TCP and UDP port scanning
- Analyze packet captures using Wireshark
- Enumerate services through NetBIOS
- Discover hosts and services using Nmap
- Understand how reconnaissance techniques are used during penetration testing

# Part 1: Discovering the Gateway Address

## Objective

Identify the default gateway used by the attacker machine.

## Command

```bash
traceroute www.smartmux.com
```

## Explanation

The `traceroute` command reveals the path packets take to reach a destination. The first hop displayed in the output represents the default gateway.

## Findings

The first hop in the traceroute output identified the gateway address used by the attacker system.

# Part 2: ICMP Packet Crafting with Hping3

## Objective

Verify connectivity to the victim machine by sending ICMP packets.

## Command

```bash
hping3 -c 3 <Victim-IP>
```

## Explanation

| Parameter | Description |
|------------|------------|
| hping3 | Packet crafting tool |
| -c 3 | Send 3 packets |
| Victim-IP | Target host |

This command sends three ICMP packets to the target machine and waits for responses.

## Results

The victim machine successfully responded to all packets, confirming that it was online and reachable across the network.

# Part 4: SYN Port Scanning

## Objective

Identify open TCP ports on the victim machine.

## Command

```bash
hping3 --scan 1-3000 -S <Victim-IP>
```

## Explanation

| Parameter | Description |
|------------|------------|
| --scan | Scan a port range |
| 1-3000 | Scan ports 1 through 3000 |
| -S | Set the SYN flag |

This technique performs a SYN scan, which is considered stealthier than a full TCP connection because the three-way handshake is not completed.

## Results

Several open ports were identified during the scan.

## Security Impact

Open ports increase the attack surface of a system and may expose services that could be vulnerable to attack.

# Part 5: UDP Packet Scanning

## Objective

Generate UDP traffic toward a target host.

## Command

```bash
hping3 <Target-IP> --udp --rand-source --data 500
```

## Explanation

| Parameter | Description |
|------------|------------|
| --udp | Send UDP packets |
| --rand-source | Randomize source IP addresses |
| --data 500 | Send 500-byte payloads |

This command generates UDP traffic with spoofed source addresses.

## Results

The target system received multiple UDP packets with randomized source addresses.

# Part 6: UDP Traffic Analysis

## Objective

## Findings

Wireshark showed:

The randomized source addresses demonstrate how attackers may attempt to obscure the true origin of UDP traffic.

# Part 8: Flood Traffic Analysis with Wireshark

## Objective

Observe flood traffic from the victim machine.

## Findings

Wireshark displayed:

- Large quantities of TCP packets
- Continuous packet transmission
- Numerous half-open connection attempts

## Analysis

The observed traffic pattern closely resembled a SYN flooding attack, which attempts to exhaust system resources by creating incomplete TCP connections.

# Part 9: NetBIOS Enumeration

## Objective

Identify services exposed through NetBIOS.

## Findings

| NetBIOS Code | Service |
|-------------|----------|
| <0x00> | Workstation |
| <0x00> | Workstation Service |
| <0x20> | File Server Service |

## Analysis

NetBIOS enumeration provides valuable information about Windows systems, including workstation identities and file-sharing services.

# Part 10: Network Scanning with Nmap

## Objective

Enumerate hosts and services using Nmap.

## Command

```bash
nmap -T4 -A -v target.example.edu
```

## Explanation

| Parameter | Description |
|------------|------------|
| -T4 | Faster timing template |
| -A | Aggressive scan |
| -v | Verbose output |

This command performs host discovery, service detection, operating system fingerprinting, and traceroute analysis.

# Key Takeaways

This lab provided hands-on experience with several foundational penetration testing techniques:

- Host discovery using ICMP
- TCP SYN scanning
- UDP scanning
- Packet crafting with Hping3
- Traffic analysis with Wireshark
- NetBIOS enumeration
- Nmap reconnaissance
- Service identification and attack surface discovery

Understanding these techniques is essential for both offensive security assessments and defensive security monitoring.
